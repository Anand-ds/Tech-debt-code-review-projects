# Tech Debt Code Review Projects

This repository provides a structured, repeatable, and transparent process for identifying, documenting, and reviewing technical debt across engineering teams.

It includes:
- Issue templates for reporting and reviewing tech debt
- A scoring model for prioritization
- A GitHub Actions workflow for automated reminders and tagging
- Documentation on how to run tech debt review cycles

## Purpose

Technical debt is inevitable, but unmanaged debt slows delivery, increases risk, and reduces developer happiness. This project helps teams:

- Identify tech debt consistently
- Prioritize based on impact and risk
- Track progress over time
- Integrate tech debt review into normal engineering workflows
