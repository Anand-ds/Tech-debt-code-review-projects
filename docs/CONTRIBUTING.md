# Contributing to Tech Debt Code Review Projects

Thank you for contributing to this repository. This project aims to create a consistent, transparent, and scalable process for identifying and managing technical debt across engineering teams.

## How to Contribute

### 1. Reporting Technical Debt
Use the **Tech Debt Report** issue template. Provide:
- Clear description
- Impact
- Severity
- Evidence or reproduction steps

### 2. Reviewing Technical Debt
Use the **Tech Debt Review** template to:
- Analyze the issue
- Provide recommendations
- Assign a priority score using the scoring model

### 3. Submitting a Pull Request
When addressing tech debt:
- Link the PR to the corresponding issue
- Follow the PR template
- Ensure tests and documentation are updated

### 4. Scoring Model
All tech debt items must be scored using the rubric in `docs/SCORING_MODEL.md`.  
This ensures consistent prioritization across teams.

### 5. Coding Standards
- Follow existing patterns and architecture
- Keep changes small and focused
- Avoid introducing new debt while fixing old debt

### 6. Communication
If unsure about anything:
- Comment on the issue
- Tag maintainers
- Propose improvements to the process

We appreciate your contributions and commitment to improving code quality.
