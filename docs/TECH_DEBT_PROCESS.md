# Tech Debt Review Process

This document outlines the lifecycle of technical debt management.

## 1. Identification
Engineers report tech debt using the "Tech Debt Report" issue template.

## 2. Triage
A weekly or bi-weekly meeting evaluates:
- Severity
- Impact
- Risk
- Priority score

## 3. Review
A reviewer completes the "Tech Debt Review" template.

## 4. Remediation
Work is planned into sprints or added to a backlog.

## 5. Tracking
Progress is monitored through GitHub issues and automated reminders.
